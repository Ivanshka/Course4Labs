------------------LAB_1-------------------
PDVGetHttpHandler.cs - обратабывает GET запросы. Принимает два параметра ParmA, ParmB.
PDVGetHttpHandler.cs - обратабывает POST запросы. Принимает два параметра ParmA, ParmB.
PDVGetHttpHandler.cs - обратабывает PUT запросы. Принимает два параметра ParmA, ParmB.
SumHttpHandler.cs - обратабывает POST запросы. Принимает два параметра X, Y и выводит их сумму.
GetPostHttpHandler.cs - обрабатывает GET и POST запросы. Если запрос GET, возвращает страницу FirstPage.html. Если запрос Post, принимает два параметра X, Y и выводит их произведение.
FirstPage.html - содержит 4,5,6 задания. Сначала выполняется SumHttpHandler из формы (<form>) с POST запросом. Тип метода указывается в аттрибуте action="POST".
Потом выполняется GetPostHttpHandler с использованием XMLHttpRequest и функцией в script formPostHandler(), которая выполняет POST запрос с введенными в поля параметрами X,Y.
Потом выполняется GetPostHttpHandler с использованием <form>, которая выполняет POST (указывается в action) запрос с введенными в поля параметрами X,Y, и получает произведение введенных чисел.
Web.config - имеет только описания обработчиков. Особое внимание аттрибутам path, verb, type!
При вводе в адресную строку пути какого-либо обработчика, программа лезет в Web.config, ищет подходящий обработчик, лезет в него и выполняет обработку.
ВАМ СТОИТ ПОМЕНЯТЬ: Названия обработчиков, пути в web.config и немного изменить FirstPage.html (стили или еще что).

ВОПРОСЫ К ЛАБЕ: 
1. Что такое ASP.NET?
ASP.NET: набор  технологий для создания web-приложений и web-сервисов компании Microsoft. 
2. Что такое http-обработчик?
Http-handler - простейшее приложение ASP.NET. Используются для генерации содержимого ответа на HTTP-запрос
3. Каким образом запросы маршрутизируются к http-обработчику?
по конфигурационному файлу web.config
